## §3 · Key test cases (按 coverage diagram 列出 must-haves)

### §3.1 · `schema.py::normalize_subtasks`

- `[REGRESSION]` 旧格式 `subtasks: ["A", "B"]` → 两个 dict · id 自动生成 · status `not-started` · 不 emit error
- 新格式完整 dict → 直接通过 · 校验 fields
- 部分 dict(只有 title)→ 补 id + 默认 status
- 混合 `["A", {id: "X", title: "B"}, "C"]` → 三个 dict · id 各自生成 · 不冲突
- 未知字段 `{id: "X", title: "Y", staus: "done"}`(typo)→ warn issue + 字段 ignore
- 空 / None / 整数(错类型)→ error issue
- id 算法 `<module-id>-<sha1(title)[:6]>`:相同 title 相同 id · 不同 title 不同 id · title 改了 id 也改(state lost · 已 documented trade-off)

### §3.2 · `schema.py::validate_subtask_schema`

- id 唯一性:重复 id 在同 module 内 → error · `Issue.reference: docs-cockpit-author · §2.4`
- status enum:`["not-started", "in-progress", "done", "blocked"]` 之外 → error
- code path 缺失 → warn(不 error)
- code 行号越界 → warn
- docs ref 不存在 → warn(走 `paths.resolve_path` 三步 fallback)

### §3.3 · `paths.py::resolve_path` + `resolve_code_anchor`

- absolute 路径直接用
- 相对 source MD 路径:source 在 `docs/spec/module/M09.md` · ref `subdir/foo.py` → repo_root + `docs/spec/module/subdir/foo.py`
- 相对 repo root:source 同上 · ref `/sourcery/worker/x.py` 或 `sourcery/worker/x.py`(三步 fallback 兜底)
- 不存在 → None
- Windows backslash:`sourcery\worker\x.py` 跟 `sourcery/worker/x.py` 等价
- code anchor 行号解析:`path:42-89` / `path:42` / `path`(无行号)三种
- **Defensive errors (from review 3A)**:
  - `OSError`(权限 / 锁)→ warn + None preview
  - `UnicodeDecodeError`(非 utf-8)→ warn + None preview
  - binary file detect(读前 1024 字节有 null byte)→ warn + None preview
  - 文件 > 5MB → warn + None preview(超大避免 read 慢)
  - 行号越界 → warn + None preview

### §3.4 · `prompt.py::render_prompt`

- 内置 4 个 template(generic / feature / fix / refactor)各渲染一次
- user override 在 `docs/prompts/<id>.md.j2`:走 ChoiceLoader · 优先用户
- template 找不到 → fallback 到 generic
- **Forward-compat (from review 2A)**:用 `{{ recent_commits | default('') }}` 守护新 vars · 现在 vars 未注入也能渲染
- SandboxedEnvironment:`{% include '/etc/passwd' %}` → SecurityError · 不会真的读
- `Loader` 白名单:仅 `docs/prompts/` + `docs_cockpit/templates/prompts/` · 其他 dir `{% include '../foo.j2' %}` → TemplateNotFound
- git 不可用(temp dir 跑)`current_branch = None` · template `{% if current_branch %}...{% endif %}` 守护
- `linked_docs` 单 doc 摘要 hard cap 2000 字符:超长截断 + `…`

### §3.5 · build.py integration

- fixture 项目:`tests/fixtures/minimal-v011/` · 1 个 `docs-cockpit.yaml` + 1 个 module MD(混旧字符串 subtasks + 新对象 subtasks)+ 1 个 concept MD + 1 个 spec MD
- 跑 `docs_cockpit.build.main(['build', '-c', fixture/docs-cockpit.yaml])`
- assert:
  - `docs/index.html` 生成 · 包含 `<script type="application/json" id="docs-data">`
  - `docs/prompts.js` 生成 · 是合法 JS · `window.__PROMPTS__` 是 dict
  - `docs/code_previews.js` 生成
  - `docs/state.json::modules[].subtasks[]` 是对象数组(string normalized · object 通过)
  - `state.json::issues[]` 包含预期的 warn / error · severity 准 · `reference` 字段指向 docs-cockpit-author 对应 §
- **`</script>` 注入测试**:fixture 里有 1 个 subtask 的 prompt rendered output 含字面串 `</script>` · build 后 `index.html` 该串被 escape 为 `<\/script>` · script tag 不破

### §3.6 · CLI integration

- `docs-cockpit prompt M09 M09-S6` → stdout 是 rendered prompt string
- `docs-cockpit prompt M09 M09-S6 --copy`:pyperclip 可用 → 复制 + stdout 仍输出 · pyperclip 不可用 → stderr 提示 `--copy disabled · pip install pyperclip to enable` · exit 0
- `docs-cockpit prompt --list` → 列 generic / feature / fix / refactor 4 个
- `docs-cockpit migrate-subtasks <file>`:dry-run 默认 · 输出 diff yaml · `--apply` 落盘 · 校验 backup file 生成
- `docs-cockpit lint --prompts`:fixture 里塞 1 个 Jinja 语法错误 template · CLI exit 1 + 输出错误位置

### §3.7 · HTML drawer JS (Playwright E2E)

- 启 `python -m http.server` 在 fixture 的 `docs/` 目录 · navigate 到 `http://localhost:PORT/index.html`
- assert subtask checklist 渲染 · code anchor 图标可见 · Copy prompt 按钮可见
- click status select 从 not-started → in-progress · reload 页面 · 状态持久(localStorage check)
- click Copy prompt:用 Playwright `page.evaluate` 注入 mock clipboard · assert 调用 + 内容是预期 prompt
- **prompts.js 缺失场景**:删 `prompts.js` 文件 + reload · Copy prompt 按钮 grey out · 不报 JS error · 主 dashboard 正常
- **`navigator.clipboard` 不可用场景**:Playwright 启用 `--disable-features=AsyncClipboard` · execCommand fallback 路径触发 · assert 仍能复制(textarea hack)

## §4 · Edge cases worth manual QA(在 Sourcery 上跑)

- 24 modules 大 cockpit 渲染速度(主 HTML 加载 < 1 秒 · prompts.js defer 加载 < 2 秒)
- bastion 49 modules 同上(预测 prompts.js ~1MB · 拖到 v0.12 拆 shard)
- macOS Safari · Windows Edge · Linux Firefox · Chrome 4 个浏览器都验证一遍 file:// 打开 + Copy prompt
- VS Code 内置浏览器 / Cursor 同上
- 用户混改 frontmatter subtask + body subtask 同一 module · 验证 id 不冲突 · drawer 不重复渲染

## §5 · Critical paths(必须 work · 否则 v0.11 不能 ship)

1. **Regression · v0.10 用户 MD 仍能 build**:Sourcery 当前 master(全部 `subtasks: list[str]`)装 v0.11 alpha · build 不报 error · cockpit 渲染正常 · 只 emit "建议升级 subtask schema" warn
2. **`__DOCS_JSON__` 不再 collision**:用户在 prompt template 里写 docs-cockpit dogfood prompt(含字面串 `__DOCS_JSON__`)· build 不破 template
3. **Copy prompt 在 file:// 工作**:cockpit 默认打开方式是双击 index.html → `file://` · execCommand fallback 必须工作 · 不能"按钮看着能点但不复制"
4. **prompts.js 缺失 graceful degrade**:不能"打开 cockpit 白屏" · 必须"主功能正常 · 只 Copy prompt grey out"

## §6 · Out of scope(本 plan 不测)

- W2 LLM 文档优化器(留 v0.12)
- MCP server 直连 Claude(留 v0.12)
- `docs-cockpit apply-patch` 自动落回(留 v0.12)
- `docs-cockpit sync-status` localStorage 合并(留 v0.11.1 patch)
- prompts.js per-module shard(留 v0.12 · v0.11 单 sidecar 够用)
